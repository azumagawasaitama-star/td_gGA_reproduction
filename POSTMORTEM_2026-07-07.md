2026-07-07 解決。約1ヶ月半（2026-05-19〜07-07）を要した問題の全体像。

対象論文: Guerci, Capone, Lanatà, PRR **5**, L032023 (2023)

## TL;DR

**単一のバグではなく、「行列の転置・複素共役・符号」の規約が式ごとに混在していたことが
原因。** 個々のズレはすべて (a) B=1 では数学的に厳密にゼロ、(b) 実対称な平衡状態では
不可視、という性質を持っていたため、「B=1 は完璧・静的計算は完璧・B=3 の動力学だけ壊れる」
という一見不可解な症状になった。正しい規約で全式を揃えると、**Λ=0 ゲージのまま拘束 F2 と
エネルギーが厳密に保存され**（解析的に証明可能）、射影・動的Λ・Baumgarte 安定化などの
対症療法はすべて不要だった。

## 1. 症状（何に悩んでいたか）

- 論文 Fig.2a: B≥3 ではクエンチ後に d(t) が急落して準定常値（~0.12）へ緩和する。
  我々の実装では B=3 が B=1 と（ほぼ）同じ非減衰振動しかしない。
- B=3 だけ F2 拘束とエネルギーがドリフト（|ΔE/E| 44〜160%、F2 → 0.1）。
- B=1 は全スキームで機械精度。静的計算も文献ベンチマークと一致。

## 2. 犯人一覧（すべて併存していた）

## 3. なぜ1ヶ月半かかったか（誤診の連鎖）

時系列で、もっともらしかったが誤りだった診断：

1. **「R_ghost≈0 だから ghost は非結合」** → fix_gauge 基底の見かけ。物理基底では R_ghost≈0.41。
2. **「PH 対称性が厳密に B 非依存を強制する」** → 論文も同条件で B 依存性を出すので不成立。
3. **「エネルギー・F2 射影で解決できる」** → 射影は d(t) を変えない行き止まり（そもそも
   正しい式系では保存が自動なので射影は不要だった）。
4. **「ゲージ混在（初期状態 Λ_eq ゲージ vs TD Λ=0 ゲージ）が拘束を破る」** → 誤診。
   正しい式系なら Λ_eq≠0 の平衡初期条件から Λ=0 で発展させても F2 は厳密保存される。
5. **共役規約の総当たり（36通り）で正解が見つからない** → 探索空間に「n 方程式の符号反転」
   「Λc の微分方向の転置」「H_emb の添字順」という軸が入っていなかった。正解は
   全軸を同時に揃えた1点なので、部分的な総当たりでは全組み合わせが「凍結」か
   「発展するがドリフト」になり、判別不能だった。

**構造的な教訓**: 規約バグが複数併存すると、1つだけ直しても悪化して見える
（例:「論文の字面通りの n 方程式に直したら d が凍結した」→ 他の式が旧規約のままだったから）。
このため「直した→悪化した→戻した」を繰り返して、正しい部分修正が棄却され続けた。

## 4. 何が決め手だったか

1. **論文 Eq.(11) の添字**: n_ab(ω) = ⟨η†_a η_b⟩ = [f(ωRR†+Λ)]_**ba** — 論文の n は
   「普通の Fermi 行列の転置」。ここから論文の全式の転置規約が連鎖的に確定した
   （Eq.17 の生成子は (RR†)ᵀ、Eq.18 の D には R*、Eq.12 の Δ は転置基底 ᵗh_s で展開、等）。
2. **F2 保存の解析的証明**: 一貫規約なら Loewner 微分の恒等式
   **[Δ, dg[M]] = [g(Δ), M]**（g=√(Δ(1−Δ))、両辺を Δ の固有基底で書くと自明）により
   d/dt(⟨f†f⟩ − ∫ρn) = 0 が厳密に成り立つ。つまり「保存しない＝規約が壊れている」の
   等価性が確立し、保存性がデバッグの検定統計量になった。
3. **コード監査**: einsum 添字・共役・転置のレベルで実装の全規約を洗い出し（サブエージェント）、
   理論の要求と1対1で突き合わせた。
4. **最後のどんでん返し（数値診断 `diag_conv.py`）**: 静的鞍点が論文の hole 規約ではなく
   **電子規約**（∫ρf(ωRRᵀ+Λ) = ⟨f†f⟩、残差 4.5e-6）で閉じていることを実測で確定。
   埋め込み状態の構成に PH 変換が織り込まれているため、論文の hole 規約はコードの
   演算子上では電子規約として現れる。hole↔電子の入れ替えで Λc の符号が反転するので、
   ここを間違えると「証明は正しいのに数値が合わない」状態になる。
   **規約は文献からの類推ではなく、必ず数値で実測して確定させること。**

## 5. 最終的に正しい式系（電子規約、コードのオブジェクトで）

```
Δe = ⟨f†f⟩ = I − ⟨op_bb⟩,   g = [Δe(1−Δe)]^{1/2}
R  = g⁻¹ ⟨f†c⟩                     （共役なし）
B  = Σ_f w_f ω_f n_f               （複素のまま。np.real 禁止）
D  = conj(g⁻¹ B R)                 （H_emb の f†c スロット係数）
M  = R Dᵀ + D* R†
Λc = +(dg_{Δe}[M])ᵀ                （Loewner 閉形式。基底ループ・least_squares 不要）
∂t Φ = −i H_emb Φ,   ∂t n = −iω[RR†, n],   Λ = 0 固定
初期条件: Λ_eq = W Λ_imp Wᵀ,  n₀ = f(ωR₀R₀ᵀ + Λ_eq)
```

## 7. 教訓

1. **保存則を検定に使う**: TDVP は正しければ拘束とエネルギーを厳密保存する。
   「dt を小さくしても直らないドリフト」は積分誤差ではなく式の誤り。
2. **規約は導出で決める**: 実数極限との一致では共役・転置・符号を判別できない。
   複素の式は複素の整合条件（保存則）から導出するしかない。
3. **総当たりは軸が揃って初めて機能する**: 多軸の規約バグは部分探索では原理的に
   見つからない。導出で軸を確定させてから探索する。
4. **「見かけの物理」に注意**: fix_gauge 後の量（R_ghost≈0 等）から物理を推論して
   誤診が生まれた。ゲージ不変量（d, E, √(D†D), eig Λc）で判断する。
5. **対症療法（射影・安定化項）は原因を隠す**: 保存が壊れているときに射影で抑えると、
   「何が壊れているか」の信号まで消える。
